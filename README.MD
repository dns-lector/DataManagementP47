# Менеджмент даних з використанням EF Core, Dapper
## Порядок встановлення
Бази даних належать до тих файлів, які ігноруються у репозиторіях.
Після клонування проєкту необхідно підключити його до наявної БД.
Рекомендується створити у директорії проєкту окрему БД:
ПКМ - Add - New item - [Show all templates]
[Data] - Service-based Database

Переіменувати файл appsettings_sample.json до appsettings.json.
Переконатись у властивостях файлу, що він копіюється до вихідної директорії.

Оновити рядок підключення у файлі конфігурації (appsettings.json)
[Рядок підключення є у властивостях БД, 
  відкритих через Server Explorer DblClk по файлу
  (не плутати з Solution Explorer)]

Запустити проєкт, вибрати "1 - Перевірити підключення",
переконатись у працездатності.

Вибрати "2 - Створити таблиці БД", переконатись, що команди виконано.

Вибрати "3 - Заповнити початкові дані", переконатись, що команди виконано. 
Дана дія заповнює таблиці-словники: Фірми, Співробітники, Товари

Вибрати "4 - Створити дані ... продажів", переконатись, що команди виконано. 
Дана дія заповнює таблицю-журнал Продажі
